Comandos UNIX:
==============

Gran parte de las herramientas usadas por JVO siguen estándares UNIX, bien sea en Windows o Linux es esencial el conocimiento sobre los comandos UNIX más básicos que permiten al usuario sacar provecho del Shell.


- Diferencias entre los sistemas UNIX y otros sistemas.
- Qué es el kernel?
- Qué es el shell?
- Interacción entre el kernel y las aplicaciones.
- Qué es un servicio/demonio?
 
Comandos básicos:
------------------------------
* cd
* ls (con sus flags) 
* man
* mv
* cp
* sort
* less
* find
* echo
* sudo

Comandos avanzados:
--------------------------------
* chmod
* chown
* grep
* egrep
* cut
* Redirecciones y pipe
(hay más comandos, investigar acerca de los otros)

- Usar el manejador de paquetes de sistemas derivados de debian (apt-get o aptitude)
- Crear un script en shellscripting
- Visualizar variables de entorno

Administración de servicios:
----------------------------------------

- Iniciar, detener y Reiniciar servicios en sistemas derivados de Debian.
- Ubicar los logs de algún servicio y filtrarlos usando los comandos anteriores.
- Ubicar, visualizar y entender los archivos de configuración de apache 2.
- Configurar un virtualhost.
- Activar o desactivar un sitio.

Manejadores de versiones:
========================

Los manejadores de versiones son esenciales para el desarrollo de software en JVO y en cualquier empresa de este estilo.

* Es necesario conocer al menos los nombres de 2 manejadores de versiones ademas de Git y sus diferencias.
* Porque es necesario un manejador de versiones y que problemas resuelve y que beneficios trae.
* Conocimiento sobre el uso intermedio de git y sus comandos: add, rm, stash, checkout, commit, pull y push.
* Que es el .gitignore
* Resolver un merge conflictivo.


Symfony2:
========

Symfony2 es un framework de desarrollo para PHP, JVO ha decidido usarlo por su escalabilidad y uso de buenas practicas en el desarrollo de software.

* Entender el tutorial de http://symblog.site90.net/index.html desde la parte 1 hasta la 3.
* Adicionalmente hacer la parte 4 y 5.
* Revisar y entender el taller de Symfony2 https://github.com/throoze/tallerSymfony2 .
* Personalizar el tutorial a gusto y montarlo en un repositorio de GitHub.
* Familiarizarse con la documentación de Symfony2 http://symfony.com/doc/current/book/index.html .
* Extra: usar Ajax con Symfony2.
* Que es serializacion de datos, para que se necesita en un sistema web y como funciona JSON.


Javascript y NodeJS:
===================

NodeJS es un sistema de software escrito en Javascript diseñado principalmente para servidores web que esta pensado para manejar comunicaciones en tiempo real entre el cliente y el servidor.

* Que es un WebSocket.
* Que es Socket.IO, como se usa y cual es su diferencia con AJAX.
* Buscar y entender algun tutorial sencillo de NodeJS donde se use Socket.IO o nowJS.
* Conocimientos intermedios sobre JQuery, manipulación de DOM, selectores.
* Familiarizarse con la documentación de Node.JS http://nodejs.org/api/
